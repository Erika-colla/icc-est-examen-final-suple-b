Examen Práctico – Evaluación de Productos

Descripción General

En un entorno comercial se requiere gestionar una colección de productos mediante operaciones que permitan:

  • Eliminar duplicados por código y ordenar los productos por nombre y código.
  • Clasificarlos según la unicidad de caracteres en su nombre.
  • Identificar productos con precios repetidos.
  • Buscar productos destacados por nombre de forma eficiente.

Estas operaciones simulan un sistema de análisis y evaluación de inventario.

Modelo: Producto

Cada objeto Producto contiene:

  • nombre: nombre del producto.
  • codigo: identificador único del producto.
  • precios: lista de precios históricos (mínimo 3 valores double).

Campos calculados

  • porcentajeCaracteresUnicos: porcentaje de caracteres únicos en el nombre, ignorando espacios y sin distinguir mayúsculas y minúsculas.
  • cantidadPreciosRepetidos: número de valores de precio que se repiten en la lista.

Ejemplo:

Nombre: "Teclado Logitech"
Sin espacios: "tecladologitech"
Caracteres únicos: {t, e, c, l, a, d, o, g, i, h} = 10
Total caracteres: 15
Porcentaje = (10 * 100) / 15 = 66

Precios: [50.0, 55.0, 50.0, 60.0]
Precios repetidos = 1

Clase a implementar: ProductoController

Método A – ordenarProducto(List<Producto> lista)

  • Retorna un TreeSet<Producto>.

  • Elimina duplicados por codigo.

  • Ordena por:

    1. nombre ascendente.
    2. codigo ascendente.

La clase Producto debe implementar Comparable<Producto> respetando esta lógica de ordenamiento.

Método B – clasificarPorUnicidad(Set<Producto> productos)

  • Retorna un Map<String, List<Producto>>.
  • Agrupa los productos según el porcentaje de unicidad de caracteres del nombre.
Rango porcentaje Categoría
90–100 A
70–89 B
50–69 C
30–49 D
0–29 E

Debe utilizar LinkedHashMap y LinkedList para conservar el orden de inserción.

Método C – obtenerDestacados(List<Producto> lista)

  • Filtra los productos con más de un precio repetido.
  • Retorna una lista ordenada por nombre en orden ascendente.

Método D – buscarPorNombre(List<Producto> lista, String nombre)

  • Aplica búsqueda binaria sobre una lista previamente ordenada por nombre.
  • Retorna el producto correspondiente o null si no existe.

Ejemplo de ejecución

List<Producto> productos = Arrays.asList(
    new Producto("Monitor Samsung", "P001", Arrays.asList(150.0, 150.0, 160.0)),
    new Producto("Teclado Logitech", "P002", Arrays.asList(50.0, 55.0, 50.0, 60.0)),
    new Producto("Mouse Logitech", "P003", Arrays.asList(30.0, 30.0, 30.0)),
    new Producto("Monitor Samsung", "P001", Arrays.asList(150.0, 155.0)),
    new Producto("Cámara Web", "P004", Arrays.asList(100.0, 100.0, 120.0)),
    new Producto("Auriculares", "P005", Arrays.asList(80.0, 90.0, 100.0))
);

Resultado esperado en consola

Productos ordenados por nombre y código (sin duplicados):
1. Auriculares (P005)
2. Cámara Web (P004)
3. Monitor Samsung (P001)
4. Mouse Logitech (P003)
5. Teclado Logitech (P002)

Clasificación por porcentaje de caracteres únicos:
A: []
B: [Teclado Logitech]
C: [Auriculares, Mouse Logitech]
D: [Monitor Samsung]
E: [Cámara Web]

Productos destacados (más de un precio repetido):
- Cámara Web
- Mouse Logitech
- Teclado Logitech

Producto buscado por nombre "Teclado Logitech":
→ Teclado Logitech (P002)

Validación automática

Las pruebas unitarias validarán:

  • Cálculos correctos de unicidad y repetición de precios.
  • Eliminación de duplicados por código.
  • Ordenamiento correcto en TreeSet.
  • Clasificación correcta en Map<String, List<Producto>>.
  • Implementación adecuada de búsqueda binaria.
  • Uso correcto de TreeSet, Set, Map y List.
